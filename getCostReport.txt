API objective: Expose a getCostReport tool response that matches the existing UI contract exactly.
Endpoint: POST /api/recon/ask
The backend receives a natural language question such as:
"Show cost"
"What is driving cost?"
"Show cost by recon"
"Show cost anomalies"
"Which recons are expensive today?"
The backend should route cost-related questions to tool:
getCostReport
Response contract:
{
  "answer": "Human readable summary of cost drivers",
  "userId": "user001",
  "sessionId": "recon-demo-session",
  "toolCalls": [
    {
      "toolName": "getCostReport",
      "toolResponse": {
        "result": {
          "businessDate": "2026-06-14",
          "generatedAt": "2026-06-14T10:30:00Z",
          "currency": "USD",
          "totalCost": 1248.72,
          "costByStage": [],
          "costByRecon": [],
          "costAnomalies": [],
          "charts": []
        }
      }
    }
  ]
}
Required result fields:
businessDate: String
generatedAt: String
currency: String
totalCost: BigDecimal

Business logic:
Return cost for all active recons for the business date, not just one recon.
Calculate totalCost as the sum of all recon costs.
Calculate percentageOfTotal for each recon.
Rank recons by descending cost.
Calculate stage-level cost totals across all recons.
Identify anomalies where:
actual cost is more than 30% above baseline, or
a failed recon has unusually high cost, or
retries/checkpoint holds increase cost materially.
Generate a concise human-readable answer.
Example answer: "Total estimated cost for today is $1,248.72 across 8 recons. PELIX is the highest cost driver at $308.10, contributing 24.7% of total cost. The main cost pressure is in POST_MATCH due to retry and checkpoint hold activity."
Important UI compatibility rules:
Keep toolName exactly as getCostReport.
Keep data inside toolCalls[0].toolResponse.result.
Do not rename costByRecon, costByStage, costAnomalies, or charts.
Return empty arrays instead of null.
Return answer even if no cost data is available.
Never fail the full response because one recon has missing cost data.
Use graceful fallback values.
